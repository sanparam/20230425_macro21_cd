일반적인 Visual Studio 문제

이 절의 내용은 Visual Studio 제품군에 포함된 모든 제품이나 다수의 제품에 적용되며 온라인 설명서보다 최신의 내용입니다.


Microsoft SQL Server는 SQL Debugging을 위하여 "시스템 계정"이 아닌 유효한 사용자 계정에서만 실행해야 함

Microsoft SQL Server에서 디버깅을 가능하게 하려면 SQL Server에 이 버전의 서버쪽 Visual Studio 구성 요소를 설치해야 합니다. 또한 Microsoft SQL Server는 유효한 사용자 계정에서 실행해야 하며 "시스템 계정"에서는 실행할 수 없습니다. "시스템 계정"에서 SQL Server는 높은 보안 수준에서 실행되며 Microsoft SQL Debugger와 같은 원격 응용 프로그램과 통신할 수 없습니다.

SQL 디버깅을 위한 SQL Server 로그인 계정을 구성하려면
  1. [시작] 메뉴의 [설정]에서 [제어판]을 눌러서 엽니다.
  2. [서비스]를 두 번 눌러서 [MSSQLServer]를 선택한 후 [Startup]을 누릅니다.
  3. "This Account"(옵션)을 누른 후 로그인 ID에 유효한 사용자 계정과 암호를 설정합니다.
  4. [OK]를 눌러서 대화 상자를 닫고 구성 내용을 저장합니다.

메모:    SQL Server를 디버깅할 때 발생하는 모든 오류는 이벤트 로그 파일에 나열되어 있습니다. 이 로그 파일을 사용하여 디버깅 과정에서 생기는 문제를 해결할 수 있습니다.


웹에서 Visual Studio 6.0 제품 등록하기

Visual Studio 6.0 제품의 설치가 끝나면 설치 마법사가 온라인 등록 페이지로 안내합니다. 설치할 때 등록할 수 없으면 나중에 웹에서 등록할 수 있습니다. 등록 사이트의 위치는 http://www.microsoft.com/isapi/redir.dll?Prd=vstudio&Pver=98&Ar=register입니다.

Windows 98에서 Visual Studio 창 보기

Windows 98에서 Visual Studio 창 제목을 보려면 화면 색상 표시를 256색보다 높게 설정해야 합니다(16비트 이상의 색상이 필요함).

Visual C++의 매크로 파일 이름 지정 시 주의 사항

Visual C++에서 매크로를 사용하는 경우, 파일 이름에 MBCS 문자를 사용하지 마십시오. 이름에 한글과 같은 MBCS 문자를 사용한 매크로의 경우, 매크로를 기록하는 중에 동작을 중단하면 프로그램이 손상될 수 있습니다. 이를 사전에 방지하는 방법은 SBCS 문자(예:영어)만을 사용하는 것입니다. 이 문제는 Visual C++5.0에서도 마찬가지로 발생합니다.

Visual InterDev에서 갤러리 예제를 만들 경우

한글판 Internet Information Server 4.0/Personal Web Server와 Visual InterDev에서 갤러리 예제를 만들 경우, 데이터베이스 관련 .asp 파일이 올바르게 작동하지 않고 다음과 같은 오류 메지시를 보여 줍니다.

ADODB.Connection error '800a0bb9'
The application is using arguments that are of the wrong type, are out of acceptable range, or are in conflict with one another.

이와 같은 경우의 해결 대안은 다음과 같습니다.

(1) 다음과 같이 MMC에서 프로젝트의 APP 루트가 올바르게 설정되어 있는지 확인합니다.

[시작] -> [프로그램] -> [NT4 OP] -> [Internet Information Server] -> [인터넷 서비스 관리자]의 Default Web Sites에서 만들어진 갤러리 프로젝트를 선택합니다. 마우스 오른쪽 클릭으로 속성을 확인합니다. -> [디렉터리] 텝의 [응용프로그램 설정]그룹에서 [시작점]이 설정되어 있지 않으면 [Create]를 클릭합니다. [시작점]이 설정되어 있지 않으면 Name 상자가 사용 불가능 상태입니다. -> [확인]를 클릭합니다.

(2) Visual InterDev 연결 속성을 다음과 같이 설정합니다.

(Project Explorer에서 노드를 찾아서)global.asa -> DataEnvironment -> 해당 연결로 가서 등록 정보를 봅니다. -> [인증] 탭에서 [디자인 모드 인증 저장]을 선택하고, 사용자 이름은 admin으로 정합니다.

Visual FoxPro의 ControlSource 속성에서 한글 입력 시 문제

Visual FoxPro 6.0에서 ControlSoruce 속성을 적용할 수 있는 컨트롤 사용 시 한글을 입력하는 데 다음과 같은 문제가 있습니다.

  • Black 커서가 지원이 안됩니다.
  • 삽입(Insert) 모드에서 한글을 입력하면 비정상적으로 입력이 됩니다.

    • 이 문제를 재현하려면:

    1. ControlSource 속성이 적용되는 컨트롤(예:textbox, listbox...)을 폼으로 가져옵니다.
    2. Property window에서 ControlSource 속성을 선택합니다.
    3. 입력란에서 한글을 입력합니다.

    두 번째 문제의 대안은 겹쳐쓰기 모드에서 한글을 입력하는 것입니다.

    Visual FoxPro의 View Designer에서 저장 대화 상자의 닫기 단추 오류

    Visual FoxPro 6.0에서 View Designer 사용 시, 저장 대화 상자의 닫기 단추를 클릭하면 오류가 발생합니다. 이 문제를 재현하려면 다음과 같이 합니다.

    1. View Designer에서 테이블을 하나 엽니다.
    2. View designer의 [Feilds] 탭에서 [Add All] 단추를 클릭하여 모든 필드를 [Selected fields]로 추가합니다.
    3. View designer를 닫기 위하여 대화 상자 오른쪽 상단의 창 닫기 단추를 클릭합니다.
    4. [Open] 대화 상자가 나타나며, [Cancel] 단추를 클릭하면 "File 'n.dbf' does not exit"라는 오류 메시지가 나타나며 Local View를 저장할 수 없습니다.

    이 문제의 대안으로는, 저장할 때 반드시 [File] 메뉴의 [Save] 또는 [Save As] 명령을 사용하는 것입니다.

    도움말 뷰어 문제

    이 절에서는 Visual Studio 6.0에 포함된 온라인 도움말 뷰어와 관련한 문제에 대해 설명합니다. Visual Studio 제품군의 도움말 시스템에 대한 내용을 보려면 MSDN™, Microsoft Developer Network Readme를 참고하십시오.

    예제 코드의 자르기와 붙여넣기가 때때로 올바르게 실행되지 않는 경우

    MSDN Library Visual Studio 설명서에서 코드 편집기로 예제 코드를 복사하여 붙여넣을 때 줄 바꿈과 서식 정보가 올바르게 복사되지 않을 수도 있습니다. 이 문제를 해결하려면 다음 중 하나를 수행하십시오.

    • 코드를 복사한 후 줄 바꿈을 수동으로 편집합니다.
    • 예제 코드의 소스를 보이게 하여 <pre>와 </pre> 태그를 포함한 예제 코드 전체를 코드 편집기로 복사한 다음에 붙여넣은 파일에서 필요 없는 부분을 삭제합니다.

    Visual Studio에 포함된 HTML Help SDK 버전을 사용자가 업데이트할 수 있음

    Visual Studio 6.0에는 HTML Help SDK 버전이 포함되어 있으며 사용자가 업데이트할 수 있습니다. 최신 버전의 HTML Help SDK를 구하려면 http://www.microsoft.com/workshop/author/htmlhelp 웹 사이트를 방문하십시오.

    ADO 개체에 대한 도움말 찾기

    예를 들어 Visual Basic에서 Recordset, Connection, Command, Parameter, ADOR, RDS 및 RDS Server 개체와 같은 ADO 개체를 사용할 경우 개체나 그 개체의 속성, 이벤트, 메서드에 대해 문맥 인식 도움말을 사용할 수 없습니다. 즉, 개체에 대한 참조를 가지고 있고 그 기능 중 한 가지를 사용할 경우 코드를 선택하고 F1을 눌러도 도움말 항목이 표시되지 않습니다. 대신에 잘못된 항목이 나타나거나 "키워드를 찾을 수 없습니다"라는 메시지가 나타납니다.

    그러나 온라인 설명서 [색인]을 사용하여 모든 개체의 속성, 이벤트, 메서드에 대한 도움말을 사용할 수 있습니다.

    1. MSDN 설명서 뷰어가 열리지 않으면 [도움말] 메뉴에서 [목차]를 누릅니다.
    2. [색인] 탭을 누릅니다.
    3. "컬렉션," "속성," "이벤트," "메서드"라는 낱말을 적절하게 포함하여 속성이나 이벤트 또는 메서드의 이름을 입력합니다.
    4. 사용할 수 있는 항목의 목록에서 제목에 "ADO"가 포함된 적절한 항목을 선택합니다.

    예를 들어 Recordset 개체의 Open 메서드에 대한 도움말을 얻으려면 "Open 메서드"를 입력한 후 목록에서 "Open 메서드(ADO Recordset)"를 선택합니다.

    메모:     MSDN Library 목차를 살펴보면 ADO 개체 모델과 같은 기타 ADO 항목에 대한 추가 도움말도 찾을 수 있습니다. 찾는 방법은 [플랫폼 SDK]를 열고 [데이터베이스 및 메시징 서비스]아래에서 [Microsoft Data Access SDK]를 선택하십시오.
