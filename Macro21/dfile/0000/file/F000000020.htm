설치 정보에는 Microsoft® Visual Studio™ 6.0 -- Windows 및 인터넷용 개발 시스템과 개발 제품군에 대한 핵심적인 내용이 들어 있습니다. 이 파일의 내용은 Visual Studio 제품군에 포함된 모든 제품이나 다수의 제품에 적용될 수 있습니다. 이 Readme는 MSDN Library Visual Studio 6.0 설명서의 내용을 업데이트한 것입니다.

설치 정보는 Visual Studio 제품군에 포함된 Readme "그룹" 중 하나입니다. 모든 제품의 Readme 파일은 Visual Studio Disk 1의 루트 디렉터리에 있습니다. 이 Readme 파일은 install.htm과 여러 개의 readmeXX.htm(여기에서 XX는 제품을 표시) 파일로 이루어져 있으며 널리 통용되는 웹 기술을 사용하여 사용자가 정보를 신속하게 찾아볼 수 있도록 하였습니다. 이 파일들은 Microsoft® Internet Explorer 4.0과 같은 웹 브라우저나 메모장에서 볼 수 있습니다.

Microsoft Internet Explorer에서 모든 Readme 파일을 한꺼번에 인쇄하는 방법은?
  1. Microsoft Internet Explorer에서 readmeXX.htm 파일이나 install.htm 파일을 엽니다.
  2. [파일] 메뉴에서 [인쇄]를 누릅니다.
  3. [인쇄] 대화 상자에서 원하는 인쇄 옵션을 설정합니다.
  4. [모든 연결 문서 인쇄]를 선택하고 [확인]을 누릅니다.

Visual Studio 프로그램 설치 지침

이 절에서는 Visual Studio 6.0에 포함된 프로그램과 파일에 있는 설치 관련 내용을 나열합니다.

Visual Studio 베타 버전, Pre-Release 및 Tech Preview 버전 위에 설치

모든 Visual Studio 제품의 Pre-Release이 Tech Preview 버전 또는 베타 버전이 설치된 컴퓨터에 Visual Studio 6.0의 최종 릴리즈를 설치하지 마십시오. 파일 복사 대상과 등록 설정은 개발 주기에 따라 변합니다. Visual Studio 제품의 Pre-Release나 Tech Preview 버전 또는 베타 버전이 설치된 컴퓨터에 최종 버전을 설치하면 일부 최종 구성 요소가 설치되지 않고 초기 베타 버전이 시스템에 남아 있게 되어 제품이 불안정해지거나 더 나쁘게는 시스템이 불안정해질 수 있습니다.

베타 테스트 플랫폼으로 사용한 컴퓨터에 정식 제품을 설치하기 전에 하드 디스크를 다시 포맷하고 운영 체제를 다시 설치하십시오.

"병행 설치": 여러 개의 Visual Studio 6.0 제품을 같은 컴퓨터에 설치

예를 들어 영문 버전의 Visual C++와 Visual Basic 6.0처럼 버전이 같고 같은 언어를 사용하는 Visual Studio 제품을 같은 워크스테이션에 별도로 설치할 수 있습니다. Microsoft는 이와 같은 설치를 지원합니다. Visual Studio 6.0 설치 마법사는 개발자의 워크스테이션에 다른 버전의 6.0 제품 계열이 설치되어 있는지 검색합니다.

일반적으로 같은 컴퓨터에서 다른 언어 버전이나 서로 다른 포인트 릴리즈(동일 버전이라도 6.0, 6.1과 같은 다른 포인트 버전이 있습니다.) 또는 서로 다른 제품 계열을 함께 사용하는 것은 지원되지 않습니다. 이것은 설치에 실패하거나, 설치에 성공하더라도 하나 이상의 Visual Studio 6.0 제품이 제대로 동작하지 않을 수 있으며, 최악의 경우에 설치한 모든 제품을 완전히 제거할 수 없다는 것을 의미합니다.

마법사가 설치 도중에 경고 메시지를 나타내는 경우에 가장 안전한 방법은 설치를 멈추고 Visual Studio 6.0 제품을 제거하는 것입니다. 검색 스키마는 다음과 같은 시나리오를 검사합니다.

  • 예를 들어 한국어 버전의 Visual Basic과 영어 버전의 Visual Basic처럼 사용자가 같은 제품의 다른 언어 버전을 설치하는 경우에는 아래와 같은 설치 조합이 지원됩니다.

    • 제품 지원되는 설치
    Visual Basic 서로 다른 두 위치에 설치합니다.
    Visual C++ 병행 설치는 지원되지 않습니다.
    Visual FoxPro 서로 다른 두 위치에 설치합니다.
    Visual InterDev 같은 위치에 설치합니다. 한 버전을 제거한 후에 다시 설치해야 합니다.
    Visual J++ 같은 위치에 설치합니다. 한 버전을 제거한 후에 다시 설치해야 합니다.
    Visual SourceSafe 같은 위치에 설치합니다. 한 버전을 제거한 후에 다시 설치해야 합니다.

    메모:    SourceSafe를 실행할 때 선택한 UI 언어는 시스템의 언어 설정을 기반으로 합니다. 해당 언어가 SourceSafe용으로 설치되지 않은 경우 영어를 대신 사용합니다.
    MSDN 기본 디렉터리에 설치합니다.

  • Visual J++ Standard Edition과 Visual InterDev Professional Edition처럼 사용자가 다른 계층의 Visual Studio 6.0 제품을 설치하는 경우에 같은 워크스테이션에 동일한 제품의 다른 계층을 설치하는 것은 지원되지 않습니다.

  • 사용자가 포인트 릴리즈를 설치하는 경우에 Visual Studio 이전 포인트 릴리즈에서 Visual Studio 최신 포인트 릴리즈로의 업그레이드를 지원합니다. 그러나 최신 버전에 이전 버전을 설치하는 것은 지원되지 않습니다.

  • 예를 들어 Visual Studio Professional 6.0에 Visual FoxPro 6.0을 설치하는 것처럼 사용자가 Visual Studio 6.0 Enterprise Edition이나 Professional Edition으로 설치된 버전에 독립형 버전의 Visual Studio 6.0 제품을 설치하는 경우에 같은 언어(영어) 버전의 Visual Studio 6.0 제품에 대한 독립형 버전의 설치가 지원됩니다. Visual Studio Professional Edition으로 Visual Basic 6.0이 이미 설치되어 있는 워크스테이션에서 독립형 버전으로 Visual FoxPro 6.0(영어)을 설치하는 것은 안전합니다.

듀얼 부트 컴퓨터에 설치시 유의 사항

듀얼 부트 컴퓨터를 사용하고 있고 두 운영 체제 모두에 Visual Studio 프로그램을 설치하여 실행하려는 경우에는 이들 제품을 별도의 디렉터리에 설치해야 합니다.

Visual C++이나 Visual Basic과 같은 Visual Studio 제품을 같은 디렉터리에 중복 설치할 경우 예기치 않은 결과가 발생할 수도 있습니다. 각 운영 체제에서 같은 위치에 제품을 설치하면 종종 디스크 공간을 절약할 수는 있지만 문제가 발생하므로 이것은 지원되지 않습니다.

예를 들어 같은 디렉터리에 제품을 설치하면 양쪽 시스템에서 제품을 완전히 제거할 수 없습니다. 시스템 하나에서 제품을 제거하면 다른 시스템에서는 시스템 파일을 제거하는 등의 예기치 않은 결과가 발생할 수 있습니다.

공유 CD 드라이브에서 Visual Studio 설치

공유 CD 드라이브에서 Visual Studio 설치하는 경우에 공유 이름은 9문자 이내여야 하며 이름에 공백이 있으면 안됩니다.

Visual Studio를 설치하기 전에 Microsoft Internet Explorer 설치 프로그램 완료

Visual Studio를 설치하는 동안 설치 마법사는 Internet Explorer 4.01 Service Pack(SP) 1을 설치합니다. Internet Explorer의 설치가 끝날 무렵 Internet Explorer 설치 마법사는 컴퓨터를 다시 시작합니다. Internet Explorer 설치 마법사는 자동으로 다시 시작되고 계속해서 업데이트된 파일을 복사하고 시스템을 구성합니다.

일부 컴퓨터에서는 Internet Explorer 설치 마법사가 이 작업을 끝내기 전에 Visual Studio 설치 마법사가 시작되는 경우도 있습니다. Visual Studio 설치 프로그램을 계속 진행하기 전에 Internet Explorer 설치 마법사가 작업을 완료하도록 해야 합니다.

특히 컴퓨터를 다시 시작하기 전에 설치 프로그램을 반드시 끝내야 합니다. Internet Explorer의 최종 구성 단계에서 컴퓨터를 다시 시작하면 Visual Studio 설치 프로그램이 손상됩니다. Internet Explorer 설치 마법사는 최종 단계에서 진행률 표시기를 보여줍니다. 이 진행률 표시기가 사라지면 Visual Studio 설치 마법사를 안전하게 실행할 수 있습니다.

Microsoft Internet Explorer 설치 프로그램에서 이전 버전의 MDAC 파일을 설치하는 경우

Microsoft Internet Explorer 4.0 설치 프로그램이 설치하는 Microsoft Data Access(MDAC) 파일은 Visual Studio 6.0 제품에서 설치하는 MDAC 파일보다 더 이전 파일일 수 있습니다. 이미 Visual Studio 6.0이 설치된 컴퓨터에 Internet Explorer를 설치할 때 이전 버전의 MDAC 파일을 설치할지 여부를 묻는 메시지가 나타나면 현재 컴퓨터의 최신 구성 요소를 그대로 유지하도록 선택합니다.

설치 마법사가 웹 게시 마법사 디스크를 찾는 경우

Visual Studio 6.0이나 Visual J++ Tech Preview 2를 설치할 때 웹 게시 마법사 디스크를 삽입하라는 대화 상자가 나타나면 대화 상자를 닫고 설치를 계속 진행합니다.

Visual Studio와 Visual J++는 모두 웹 게시 마법사를 별도의 디스크로 제공하지 않습니다. 사용자가 Iomega Jaz 드라이브와 같은 이동식 드라이브에 Temp 디렉터리를 설정한 컴퓨터에서 Visual Studio나 Visual J++를 설치할 때 이와 같은 메시지가 나타납니다.

Visual Studio나 Visual J++ 설치를 완료한 후 웹 게시 마법사를 설치하려면 아래와 같이 수행합니다.

  1. Temp 환경 변수를 하드 드라이브와 같은 고정 매체의 디렉터리로 설정합니다.
  2. CD의 [CD Drive:] \setup에서 wpie15.exe를 실행합니다.
  3. 웹 게시 마법사 설치를 완료한 후 필요한 경우에 Temp 환경 변수 디렉터리를 이전 디렉터리로 재설정합니다.

Visual Studio 6.0 제품을 설치한 후 클라이언트 도구와 서버 구성 요소 설치

설치 과정은 몇 가지 요소로 이루어집니다. 이들 구성 요소는 [제어판]의 [프로그램 추가/제거]에서 실행할 수 없습니다. [프로그램 추가/제거]를 사용하면 제품 설치에 필요한 설치 부분만 얻을 수 있습니다.

설치 마법사를 사용하여 Windows NT 서버 컴퓨터에서 Visual Studio 6.0 서버쪽 구성 요소를 설치할 때 "다른 설치 프로그램 사본이 이미 실행중입니다. 다른 설치 프로그램을 완료하거나 Windows를 다시 시작하십시오."라는 메시지가 나타날 수 있습니다.

이러한 상황은 대개 ACME 설치 프로그램이 끝나기를 기다리는 동안 마법사 타이머가 종료되는 느린 컴퓨터에서 발생하며 마법사는 다음 설치를 시작합니다.

메시지 상자에서 [확인]을 누릅니다. 이렇게 하면 마법사가 충돌하는 프로세스중 하나를 종료하고 남은 설치를 계속하도록 할 수 있습니다. 설치 종료시 자동으로 생성되는 요약 페이지에는 구성 요소가 완전하게 설치되지 않은 경우에도 구성 요소의 목록이 나열됩니다.

설치에 실패한 특정 구성 요소를 다시 설치하려면 설치 마법사를 다시 실행합니다.

Windows 95에서 서버쪽 구성 요소 설치

모든 서버 응용 프로그램은 Windows NT를 실행하는 컴퓨터에 설치하는 것이 좋습니다.

Visual Studio 6.0 Enterprise Edition과 Professional Edition은 특별히 Visual Studio 개발자에게 알맞은 다양한 Micosoft 서버 응용 프로그램을 패키지로 만든 제품입니다. 대부분의 응용 프로그램은 Windows 95나 Windows NT 운영 체제에서 설치할 수 있습니다. 모든 서버 응용 프로그램은 Windows NT 서버나 Windows NT 워크스테이션에 설치하는 것이 좋습니다.

Windows 95 컴퓨터에 서버를 설치하도록 선택한 경우에는 설치 순서가 중요합니다. Windows 95에서 제품을 성공적으로 설치하고 제대로 동작하도록 하려면 일부 응용 프로그램이 다른 프로그램보다 먼저 설치되어 있어야 합니다. 아래의 순서에 따라 구성 요소를 설치합니다.

  1. NT Option Pack
  2. FrontPage 98 Server Extensions
  3. Data Access Components 2.0
  4. Visual InterDev Server Components
  5. 응용 프로그램 성능 탐색기
  6. Visual Studio Analyzer
  • (순서에 관계 없이) Visual Source Safe Server
  • (순서에 관계 없이) Visual FoxPro Server Samples
설치 마법사는 같은 순서로 이 옵션을 제공합니다. 다시 말해서 Windows 95에서는 순서가 중요합니다.

Windows 95 컴퓨터와 관련된 문제는 Windows 9x 컴퓨터에서의 문제를 참고하십시오.

SQL Server 6.5와 Windows NT Option Pack 4.0의 알려진 호환성 문제

Windows NT Option Pack 4.0이 설치된 컴퓨터에 SQL Server 6.5를 설치하는 경우 일부 디자인 모드 컨트롤(DTC)을 덮어쓸 수도 있습니다.

이러한 상황이 발생하면 Microsoft Transaction Server(MTS)를 시작할 때 오류 메시지가 나타나면서 MTS를 다시 설치하라고 지시합니다.

이러한 상황을 방지하려면 SQL Server를 설치한 후 NT Option Pack을 설치합니다.

Windows NT Symbols Setup에서 발생하는 오류 메시지

사용자가 Windows NT 4.0에서 "Windows NT Symbols Setup"을 수행하여, "dbg" 파일들을 설치하고자 할 때 "comctl32.dbg"와 "oleaut32.dbg"파일이 제대로 인식되지 않아 다음과 같은 오류 메시지가 나타납니다.

The version of the file '' does not match the corresponding dll on your machine. Do you want to copy?

이 때 메시지의 내용을 무시하고, 계속 진행하면 제대로 설치됩니다.

디버깅을 위한 구성

이 절에서는 디버깅 서비스에 대하여 컴퓨터를 디버깅하고 구성하는 문제에 대해 설명합니다.

ASP 디버깅을 위해 BackOffice 설치 후 IIS 패치 설치

Internet Information Server(IIS)는 Visual InterDev에서 Active Server Pages(ASP)의 디버깅과 관련된 문제를 수정하는 패치가 있습니다. 이 패치는 또한 웹 서버를 중지하고 다시 시작할 때까지 데이터 연결의 추가, 변경, 제거와 같은 Global.asa에 대한 변경 사항이 응용 프로그램에 적용되는 Windows 95/98 문제를 해결할 수 있습니다.

ASP 디버깅이 올바로 동작하는지 확인하거나 Windows 95/98 문제를 해결하려면 BackOffice 설치 마법사를 완료한 다음 이 패치를 적용합니다.

Windows NT 컴퓨터에 IIS 패치를 적용하려면

  1. 설치 디스크에서 \IIS\WinNT\ASP2FIX1.exe를 실행합니다.

    2. 제품 디스크 위치
    Visual Studio, Professional Edition CD#2
    Visual Studio, Enterprise Edition CD#3
    Visual InterDev CD#1

  2. 프롬프트가 나타나면 다시 시작합니다.

Windows 9x 컴퓨터에 IIS 패치를 적용하려면

  1. 명령 프롬프트에서 pws /stop을 입력하고 <Enter>키를 눌러서 웹 서버를 중지시킵니다.

  2. \<windows>\system\inetsrv 디렉터리에서 Asp.dll의 이름을 Asp.old로 바꿉니다.

  3. 설치 디스크의 \IIS\Win9x\ 디렉터리에서 Asp.dll을 복사한 다음 컴퓨터의 \<windows>\system\inetsrv 디렉터리에 붙여넣습니다.
    4. 제품 디스크 위치
    Visual Studio, Professional Edition CD#2
    Visual Studio, Enterprise Edition CD#3
    Visual InterDev CD#1

  4. 명령 프롬프트로 복귀하여 pws /start를 입력하고 <Enter>키를 눌러서 웹 서버를 다시 시작합니다.

SQL Server 6.5 및 Visual C++ 5.0 이상이 설치된 컴퓨터에서 SQL Debugging 서비스 사용

SQL Server 6.5 및 Visual C++ 5.0 이상이 설치된 컴퓨터에서 SQL Debugging 서비스를 사용하려면 컴퓨터의 원격 자동화(RA) 구성 속성이 올바로 설정되어 있는지 확인합니다.

RA 구성 속성을 확인하려면

  1. dcomcnfg.exe를 실행합니다.
  2. 'Remote Automation CLSID_StubMarhsaler'를 선택합니다.
  3. [Properties]을 누릅니다.
  4. [Identity] 탭에서 'Interactive User'를 선택합니다.

또한 SQL 디버깅 시나리오는 로컬 시스템 계정에서 서비스로 SQL Server를 실행할 경우에는 작동하지 않을 수도 있습니다. 반드시 [This Account]을 선택하고 유효한 계정과 암호를 입력해야 합니다.

Windows 9x 컴퓨터에서의 문제

이 절에서는 Visual Studio 6.0을 Windows 95나 Windows 98 컴퓨터에 설치할 때 나타나는 문제에 대하여 설명합니다.

Windows 95 레지스트리 크기: 64KB의 한계로 인해 Visual Studio 설치시 경고 메시지가 나타남

Windows 95 시스템 레지스트리에서 SharedDLL 키의 크기가 64KB로 제한되어서 Visual Studio 6.0을 설치할 때 경고 메시지가 나타날 수 있습니다.

설치를 계속하기 전에 Visual Studio를 설치할 설치 경로를 줄여서 필요한 레지스트리 공간을 줄여야 합니다.

또한 [제어판]에서 [프로그램 추가/제거]를 사용해서 더 이상 사용하지 않는 프로그램을 제거할 수 있습니다. 이 프로그램에 독점적으로 사용되는 레지스트리 키는 삭제됩니다.

보관할 프로그램에 사용되는 레지스트리 공간을 더 줄이려면 프로그램을 제거한 다음 더 짧은 설치 경로로 프로그램을 다시 설치합니다.

레지스트리에서 필요 없는 항목을 정리하기 위하여 만들어진 RegClean 유틸리티를 사용할 수도 있습니다. 자세한 내용을 보거나 다운로드하려면 http://support.microsoft.com/support/kb/articles/q147/7/69.asp 사이트를 방문하십시오.

메모:    Windows NT 워크스테이션과 서버에서는 Visual Studio 설치 프로그램이 자동으로 레지스트리 크기를 변경합니다.

Visual Studio 6.0 설치 프로그램을 실행하기 전에 자동 로그온 기능 해제

자동 로그온 기능이 구성되어 있는 Windows 95 컴퓨터에서는 Visual Studio 설치 프로그램이 실행되지 않습니다.

자동 로그온 기능이 설정된 Windows 95 컴퓨터에서 Visual Studio 6.0이나 기타 제품을 설치하기 전에 http://support.microsoft.com/support/kb/articles/q141/8/58.asp에 있는 지침을 수행해서 자동 로그온 기능을 해제해야 합니다.

메모:    여기에는 사용자의 시스템에 관련되지 않을 수 있는 몇 가지 추가 단계가 있습니다.

자동 로그온 기능이 구성되어 있는 Windows 95 컴퓨터에서 Visual Studio 설치에 실패한 경우에는 자동 로그온 기능을 해제하고 설치 단계를 반복합니다.

Windows나 다른 응용 프로그램을 시작할 때 인터넷 연결 대화 상자가 나타남

Windows 95나 Windows 98에서 원격 연결이 가능하면 시스템은 Windows를 시작하거나 다른 많은 응용 프로그램을 시작할 때 인터넷 연결을 시도합니다. 이런 기능을 종종 "자동 다이얼"이나 "자동 연결"이라고 합니다. 레지스트리 키 EnableRemoteConnect를 "N"으로 설정해서 이 기능을 해제할 수 있습니다.

이렇게 하려면 아래 세 줄이 정확히 들어 있는 텍스트 파일 "DisbleAutoConnect.reg"를 작성합니다.

REGEDIT4
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\OLE]
"EnableRemoteConnect"="N"

원격 연결을 사용할 수 없도록 이 설정을 변경해도 평소처럼 인터넷을 사용할 수 있으며 이 설정값은 대부분의 시스템에서 기본값입니다. 그러나 일부 DCOM 기능에서는 원격 연결 사용이 필요합니다(http://support.microsoft.com/support/kb/articles/Q177/3/94.asp 참고).

원격 연결이 사용 가능해야 하는 경우에는 아래 세 줄이 정확히 들어 있는 둘째 REG 파일, "EnableRemoteConnect.reg"를 작성합니다.

REGEDIT4
[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\OLE]
"EnableRemoteConnect"="Y"

"EnableAutoConnect.reg"를 실행해서 자동 연결이 가능하도록 설정한 다음 자동 연결이 더 이상 필요하지 않을 때에는 "DisbleAutoConnect.reg"를 실행해서 자동 연결을 해제합니다.

Windows 95 컴퓨터에 PWS를 설치한 후 설치 마법사를 다시 시작할 수 없는 경우

Windows 95 컴퓨터에 Personal Web Server(PWS)를 설치하도록 선택하면 컴퓨터를 다시 시작한 후에 설치 마법사가 제대로 재개되지 않을 수도 있습니다. 계속 진행하려면 Visual Studio 6.0의 첫째 CD에 있는 Setup.exe를 다시 시작합니다. 설치 마법사는 자동으로 적당한 지점에서 설치 프로그램을 다시 시작합니다.

일부 Windows 95 컴퓨터에 Microsoft FrontPage 98 Extensions 설치

설치 마법사가 다른 클라이언트나 서버 설치 프로그램을 시작하면 마법사는 모습을 나타내지 않고 구성 요소 설치가 끝날 때까지 기다립니다. 마법사가 다시 나타나면 사용자는 다른 구성 요소를 설치할 수 있습니다.

일부 Windows 95 컴퓨터에서 Microsoft FrontPage 98 Extensions을 설치할 때 설치 마법사는 FrontPage 설치가 완료된 후 몇 분 동안 나타나지 않습니다.

마법사가 나타나지 않는 동안에는 설치 프로그램을 다시 시작할 수 없습니다. 마법사가 이미 실행되고 있다는 것을 프로그램이 탐지하면 다음 설치 마법사는 시작되지 않습니다.

이미 설치된 다른 프로그램에 미치는 영향

이 절에서는 컴퓨터에 이미 설치된 프로그램에 영향을 미칠 수 있는 설치 문제에 대해 나열합니다.

Microsoft Exchange나 Microsoft Outlook이 미리 설치되어 있지 않으면 IISAM Exchange 드라이버를 실행할 수 없습니다.

IISAM Exchange 드라이버는 컴퓨터에 Visual Studio를 설치하기 전에 Microsoft Exchange나 Microsoft Outlook이 완전히 설치되어 있어야 합니다. 두 가지 모두 설치되어 있지 않으면 IISAM Exchange 드라이버가 자동으로 등록되지 않습니다.

Visual Studio 6.0을 설치한 후에 문제를 해결하려면 Microsoft Exchange나 Microsoft Outlook을 설치한 다음 아래의 명령줄을 사용하여 IISAM Exchange 드라이버를 수동으로 등록하면 됩니다.

Regsrv32 msexch35.dll

Microsoft Office 97을 설치 삭제한 경우

Microsoft Office 97을 설치 삭제한 경우, 다음과 같은 문제가 발생할 수 있습니다.

  • 비주얼 구성 요소 관리자(VCM)이 올바르게 작동하지 않습니다.
  • MDAC과 관련된 일부 파일들이 함께 삭제될 수 있습니다. 따라서 비주얼 구성 요소 관리자의 "지역 데이터베이스" 트리를 사용자가 확장하려고 하는 경우, 다음과 같은 오류 메시지를 만날 수 있습니다.
    • [Microsoft][ODBC Microsoft Access 97 driver] 일반 오류. 지정한 리소스 이름을 이미지 파일에서 찾을 수 없습니다.

    이 문제를 해결하려면 MDAC_TYP.EXE를 설치하거나 Visual Studio 98을 다시 설치해야 합니다.

    Data Access SDK 2.0

    이 절에서는 Data Access SDK 2.0과 관련된 모든 문제에 대하여 논의합니다.

    설치 요구 사항

    Visual Studio와 함께 제공된 Data Access SDK를 설치하기 전에 Microsoft Data Access 구성 요소가 먼저 설치되어 있어야 합니다. 그렇지 않은 경우 SDK 설치 프로그램을 실행할 때 아래와 같은 메시지가 나타납니다.

    ODBC is not installed on this machine. If you continue, you will receive errors. Do you wish to continue anyway?

    이 문제를 해결하려면 반드시 Data Access SDK를 설치하기 전에 Visual Studio 설치 프로그램을 실행해야 합니다.

    Microsoft Data Access SDK 2.0 설치 방법

    Microsoft Data Access SDK 2.0은 2번 디스크의 \dasdk 디렉터리에서 DASDK.exe 파일을 실행하여 설치할 수 있습니다.

    SDK에는 Microsoft Data Access 구성 요소 예제와 ODBC 및 OLE DB 개발자를 위한 기타 정보와 도구들이 포함되어 있습니다.

    SQL Server 6.5나 그 이전 버전을 MDAC 2.0과 함께 사용하려면 카탈로그 프로시저를 업데이트해야 합니다. 자세한 내용은 SQL Server 드라이버 도움말 파일인 drvssrvr.hlp를 참고하십시오.
